# Drive Source Register

This is the public-safe source register for the launch repository.

Google Drive remains the authoritative source archive for original PDFs, DOCX files, drafts, and supporting materials. This public GitHub repository stores curated, versioned, implementation-ready derivatives only.

The filename-level Drive inventory is intentionally not published here because this repository is public. Keep exact filenames, private context, access notes, and sensitive source status in the protected Drive workspace or another access-controlled index.

## Source Categories and Roles

| Source Category | Role in Launch | Public Handling |
| --- | --- | --- |
| Ecosystem launch planning material | Strategic implementation blueprint for the launch. | Convert into architecture, release, and project handoff docs. Do not publish raw planning material by default. |
| Walyalup Bridge story material | Primary public entry point and testimony-first publication. | Publish only approved public text preserving authorship, dignity, and context. |
| Author and closing-note material | Author page, context note, closing note, and publication support. | Use after review for tone, accuracy, and public readiness. |
| TimeShift Fremantle / AR strategy material | Future-facing civic, cultural, creative, and place-based participation layer. | Use for interest capture and future partner-facing material after claim review. |
| Diamond Soul / OmniStrux architecture material | Deeper framework, systems architecture, and future expansion context. | Use for internal architecture and later opt-in deeper context, not first-launch headline copy. |
| Return Charter / developer continuity material | Governance spine and non-negotiable release standards. | Adapt into governance files, release gates, and implementation rules. |

## Public Repository Rules

- Preserve original source files in Drive.
- Keep raw source, working derivatives, and public releases separate.
- Do not publish sensitive, unresolved, or unapproved material.
- Do not imply partnership, endorsement, funding approval, or official permission unless documented.
- Use clear, practical, emotionally grounded public language before introducing deeper architecture.

# Public Source Register

Google Drive remains the authoritative source archive for original PDFs, DOCX files, drafts, and supporting materials. This public repository stores curated, versioned, implementation-ready derivatives only.

This register intentionally uses source categories rather than private Drive filenames. The filename-level inventory should remain in the protected Drive workspace or another access-controlled working index.

| Source Category | Role | Public Handling |

| Diamond Soul / OmniStrux architecture materials | Deeper conceptual and systems architecture. | Use for internal architecture and later opt-in deeper context, not first-launch headline copy. |
| Return Charter and developer continuity materials | Governance spine and developer continuity reference. | Adapt into governance files, release gates, and implementation rules. |

## Handling Rules

- Preserve original files in Drive before editing, redacting, or publishing.
- Keep raw source, working derivatives, and public releases separate.
- Do not publish sensitive, unresolved, or unapproved material.
- Do not imply external partnership, endorsement, funding approval, or official permission unless documented.
- Public language should be clear, practical, and emotionally grounded before introducing deeper architecture.
